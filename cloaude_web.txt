Here's the structure that makes bulk muting actually feasible.
WoW's sound folder hierarchy
Sound files live under predictable paths. The main top-level folders are:

Sound/Creature/ — every NPC and creature noise (organized by creature name)
Sound/Character/ — playable race sounds, footsteps, jump grunts, breathing, death cries
Sound/Spells/ — spell casts, impacts, fizzles, channels
Sound/Item/ — weapon swings, hits, armor foley, gear pickup
Sound/Doodad/ — environmental object sounds (doors, chairs creaking, anvils, water wheels) — this is what "ambient" doesn't catch
Sound/Music/ — music tracks
Sound/Interface/ — UI clicks, bag open, error blips
Sound/Emitters/ — placed ambient emitters in the world

Inside each, subfolders group by category. For example:

Sound/Item/Weapons/Sword/ — all sword swing variants
Sound/Item/Weapons/Bow/ — bow draws and releases
Sound/Character/NightElf/Female/ — every NElf female vocalization
Sound/Spells/Fizzle/ — all the fizzle noises in one folder

The bulk-mute approach
The straightforward path: use MuteSoundFile (still actively updated, works on Classic/Anniversary clients) plus a pre-made list. Two good shortcuts so you don't compile thousands of IDs yourself:
1. "Mute WoW Sounds" (formerly "Mute Annoying WoW Sounds") — a companion addon that ships pre-built category bundles you can toggle on/off: mechanical mounts, biological mount footsteps, hunter/mage/warlock pets, fizzles, battle pet idle loops, mage portal/armor sounds, etc. Caveat from the search: it's marketed as Retail-only since 8.2, so it may not work on Anniversary out of the box — worth installing and testing, since the underlying API works in TBC clients.
2. Pre-made Lua lists — people who've done this for sensory reasons post their full ID lists on the Blizzard forums and GitHub. You drop them into a tiny personal addon or a /run MuteSoundFile(id) macro chain. The wiki shows the format:
lualocal sounds = {
  569772, -- sound/spells/fizzle/fizzleholya.ogg
  569773, -- sound/spells/fizzle/fizzlefirea.ogg
  -- etc
}
for _, id in ipairs(sounds) do MuteSoundFile(id) end
3. Browse by path on wow.tools — go to wow.tools/files and search by path prefix like sound/item/weapons/ or sound/character/ to grab whole categories of FileIDs at once. Tedious but exhaustive.

For anything still bothering you, use the Sound ID Finder addon — it logs every sound file as it plays in real time, so when something annoys you, you grab its ID from your chat window and add it to your mute list.

The "play, identify, mute" loop with Sound ID Finder is honestly the most efficient way once your base bundles are in — you only mute what actually triggers you instead of guessing.
One thing worth flagging from the wiki: mutes don't persist across game restarts unless they're in an addon's saved settings — so make sure whatever solution you use stores them in a SavedVariables file, not just runtime macros.
